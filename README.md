# ✈️ TripMe - Sua Próxima Aventura Começa Aqui

## 📄 Descrição do Projeto

Este projeto consiste em uma landing page responsiva para uma agência de viagens fictícia, a "TripMe". O objetivo principal é apresentar um destino europeu encantador e remoto, atraindo potenciais clientes com um design limpo e uma navegação intuitiva. A página foi desenvolvida com foco em semântica HTML e estilização CSS, sem a necessidade de JavaScript para suas funcionalidades básicas.

## ✨ Funcionalidades

*   **Banner Imersivo:** Um banner de destaque com uma imagem de um pitoresco vilarejo europeu e um texto de chamada (overlay) convidativo.
*   **Navegação Clara:** Uma barra de navegação (`<nav>`) com links internos para as principais seções da página: "Home", "TripMe", "Nos Encontre" e "Conselhos".
*   **Seções Informativas:**
    *   **TripMe:** Apresenta a proposta da agência e seus diferenciais através de cards de serviço.
    *   **Nos Encontre:** Informações de contato e localização da agência.
    *   **Conselhos:** Dicas úteis para viajantes que desejam explorar destinos remotos.
*   **Interatividade com CSS:** Efeitos de `hover` nos links da navegação, botões de CTA (Call to Action) e cards de serviço, proporcionando uma experiência de usuário dinâmica.
*   **Semântica HTML5:** Utilização de tags semânticas (`<header>`, `<main>`, `<section>`, `<footer>`, `<article>`) para uma estrutura de documento clara e otimizada para SEO e acessibilidade.
*   **Responsividade:** Design adaptável para garantir uma boa visualização em diferentes dispositivos (desktops, tablets e smartphones).

## 🚀 Tecnologias Utilizadas

*   **HTML5:** Para a estrutura e conteúdo da página.
*   **CSS3:** Para a estilização e responsividade.

## 🛠️ Como Rodar o Projeto Localmente

Siga os passos abaixo para configurar e visualizar o projeto em sua máquina local:

1.  **Clone o Repositório:**
    ```bash
    git clone <URL_DO_SEU_REPOSITORIO>
    cd tripme-landing-page
    ```

2.  **Abra o Arquivo:**
    Simplesmente abra o arquivo `index.html` em seu navegador de preferência. Não é necessário um servidor web para as funcionalidades básicas, mas você pode usar um para simular um ambiente real (ex: `python -m http.server` ).
